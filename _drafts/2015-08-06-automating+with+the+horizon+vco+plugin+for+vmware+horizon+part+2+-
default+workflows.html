In this post we'll connect the vCO plugin with our View Pod and run some workflows. If you're looking to install the plugin, this was covered in Part 1.

Picking up where Part 1 left off, we need to configure the vCO connection to our View Pod. According to VMware pubs, we'll need to ensure the following prereqs are in place:

  • Ensure the vCO server can resolve the FQDN of the vCenter Server

  • Ensure you have an administrator account that's a member of a vCO Admin Security Group in AD. This account will authenticate through vCenter SSO.

  • Ensure you have View Admin credentials


Connect vCO Plugin to your View Pod


  1. Login to the Orchestrator Client and click the blue Workflows Icon

  2. Navigate to Library > Horizon > Configuration > View Pod Configuration

  3. Right-click Add View Pod in Configuration and choose Start Workflow

  4. Fill out the appropriate fields.

  5. Verify the Workflow is green and is marked as completed
