However when it comes to vROPs for Horizon (V4H), some things require a bit more intervention to fine-tune the interactions between your Virtual Desktop Infrastructure and vROPs.

For example, if using Instant Clone technology for your Horizon Desktops, you might find alerts in vROPs showing VM Disk IO Latency caused by snapshots on your CP-Parent-* VMs. In most situations, this is benign given the nature of the Instant Clone architecture.

So how do we deal with these unnecessary messages in vRealize Operations? Enter Custom Object Groups. You can follow the official documentation here.
